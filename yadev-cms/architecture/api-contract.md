## Conventions

- **Base URL:** `/v1/` for versioned API. `/health` is unversioned by design.
- **Auth column** values:
  - `Public` — no authentication required.
  - `Public (rate-limited)` — login throttle (5/min IP, 3/min email).
  - `Bearer (Sanctum)` — requires `Authorization: Bearer {token}` header. Token issued by `POST /v1/auth/login`.
  - `Bearer + tenant scope` — Sanctum bearer **plus** the token must carry an ability matching `tenant:{tenant_id}:*` (or `*` for super-admins). Enforced by `tenant.path` middleware.
  - `HMAC X-YaDev-Signature` — Runner → API callback. Authenticated by HMAC-SHA256 of the raw body using `RUNNER_CALLBACK_SECRET`. NO Sanctum.
- **Cross-tenant safety:** any id that does not belong to the active tenant returns 404, never 403 — see `architecture/security-model.md` §1.
